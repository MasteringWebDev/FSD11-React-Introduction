# React 
  - A JavaScript library used to build UI (User interfaces)
  - A webpage is built using multiple "components"
  - Introduced by Facebook (Meta) in 2013
  - Documentation: https://react.dev/

  - Declarative Library
    - We describe what the UI should look like
    - React figures out how to "update the DOM efficiently"
  
  - React efficiently updates the UI
    - Virtual DOM: Lightweight JS representation of the Real DOM
      - Diffing: Comparing the old Virtual DOM with the new Virtual DOM to identify the changes
      - Reconciliation: React compares changes and updates the Real DOM efficiently
      - Reference image: https://media.geeksforgeeks.org/wp-content/uploads/20241228111254637293/Diffing-Algorithm.jpg

  - SPAs (Single Page Application)
    - Browser loads a "single HTML file"
    - React updates the UI without full page reload
    - Reference image: https://www.hestabit.com/blog/wp-content/uploads/2021/08/re-create-1024x536.jpg

  ---------------------------------------------
  - Components:
    - Building block for React application
    - Small reusable pieces of UI
    - Features:
      - Can be reused
      - Can be nested
    - Function component syntax:
      function Component1() {
        return (
          <h1>Component 1</h1>
        )
      }
      export default Component1
      - Naming convention: PascalCase
      - Good practice: Keep filename and ComponentName same
    - Components should return "JSX"
    - To use a component in another file
      - Import the component
      - Syntax:
        <Component1 />
        <Component1></Component1>

    - JSX:
      - JavaScript XML Syntax
      - Allows writing HTML-like syntax inside JavaScript
      - Gets converted into JavaScript by React tooling
      - Rules:
        - Use className instead of class
          <div className='test'></div>
        - Self-closing tags either require / in the end, or a closing tag
          <img />
          <img></img>
        - Use {} to display variables/expressions
          - Eg.: <h1>{name}</h1>
          - true, false, null, undefined do not render anything
          - Comment: {/* */}

    - Style in React.js
      1. Create an external .css file and import this file in the component file
      2. Create a style object with properties and values and assign it to 'style' prop

    - Props ***:
      - Makes components dynamic
      - Data passed from the parent component to child component (One way data flow)

      - Pass from parent component:
        <Child1 prop1={value1} prop2={value2} />

      - Accept prop in child component
      function Child({ prop1, prop2 }) {
        return (
          <h1>Child</h1>
          <p>{prop1}</p>
          <p>{prop2}</p>
        )
      }

    - Conditional Rendering ***:
      - Displaying different UI based on a condition
      - Using Logical AND (&&) ***
        {isPremium && (<h1>Premium User</h1>)}
        - Use this when JSX needs to be shown only when condition is true
      - Using Ternary Operator ***
        {isPremium 
          ? (<h1>Premium User</h1>)
          : (<h1>Non-Premium User</h1>)
        }
        - Use this when two possible JSX outputs need to be shown for true/false
      - if else
        - Use this when two possible "large UI channges JSX" need to be shown for true/false

    - Rendering Lists ***
      - Display multiple elements from an array
      - Usually done using map()
        - Iterated through array
        - Creates JSX for every element
        - Returns a new array (Array of JSX elements)
      - Syntax:
        {arr.map((element, index) => (
          <h1 key={index}>{element}</h1>
        ))}
      - Ensure each child has a unique 'key'
        - React requires a unique key for each rendered element
          - Helps react identify elements uniquely
          - Improves reconciliation performance

    - Render: React executes the component function and determines what UI should be displayed (JSX -> HTML loaded in the broswer)

    - React component can have 2 types of data ***:
      1. Props: Data received from the Parent component
      2. State: Data belonging to that component
      - Rules:
        - Component re-renders whenever there is any change in props/state
        - State value should only be updated using its stateHandler

    - Event Handling: Responding to user actions
      - Example
        function handleClick() {
          console.log('This was clicked')
        }
        <button onClick={handleClick}>Click Me</button>
      - Event handler changes in React JSX:
        - Even handler should be in camelCase (onClick)
        - {} instead of ""
        - Function reference instead of Function call
      - If argument needs to be passed
        - Use a callback
        - Example:
          function handleClick() {
            console.log('This was clicked')
          }
          <button onClick={() => handleClick(1)}>Click Me</button>
      
    - State ***:
      - Data owned by a component
      - Used when data changes over time
      - Updating state causes component re-render

      - To create a state, 'useState' hook is used (Import this from 'react')
      - Syntax: const [state, stateHandler] = useState(defaultValue)
        - State value should only be updated using its stateHandler
        - state: Component's own data
        - stateHandler: Function used to update state
        - defaultValue: Default value for the state
      - Rules:
        - Never update the state directly (use stateHandler)
        - Component re-renders on state changes automatically if done using stateHandler

      - State Handler is asynchronous in nature
        - State updates are scheduled by React
        - State does not update synchronously/immediately
        - Hence, whenever the next state handler needs the updated state from the previous state handler, use callback syntax
          - stateHandler((prevState) => prevState + 1)

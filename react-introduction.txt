# React 
  - A JavaScript library used to build UI (User interfaces)
  - A webpage is built using multiple "components"
  - Introduced by Facebook (Meta) in 2013
  - Documentation: https://react.dev/

  - Declarative Library
    - We describe what the UI should look like
    - React figures out how to "update the DOM efficiently"
  
  - React efficiently updates the UI
    - Virtual DOM: Lightweight JS representation of the Real DOM
      - Diffing: Comparing the old Virtual DOM with the new Virtual DOM to identify the changes
      - Reconciliation: React compares changes and updates the Real DOM efficiently
      - Reference image: https://media.geeksforgeeks.org/wp-content/uploads/20241228111254637293/Diffing-Algorithm.jpg

  - SPAs (Single Page Application)
    - Browser loads a "single HTML file"
    - React updates the UI without full page reload
    - Reference image: https://www.hestabit.com/blog/wp-content/uploads/2021/08/re-create-1024x536.jpg

  ---------------------------------------------
  - Setup
    1. Download Node.js 
      - https://nodejs.org/en/download

      - NPM (Node Package Manager)
        - Used to install React and other libraries
        - https://www.npmjs.com/
      
      - Verify:
        - node -v
        - npm -v

      - Why Node.js?
        - React runs in the browser
        - Node.js is required to run development tools such as Vite and NPM
      
    2. Use Vite to create a new React project
       - Vite: A frontend build tool that helps
        - Create react project
        - Start development server
        - Convert JSX into JavaScript
        - Documentation: https://vite.dev/

      - Command: npm create vite@latest
        - Project: react-intro
        - Framework: React
        - Variant: JavaScript
        - First time:
          - Start development server right after install
        - Second time and onwards:
          - cd react-intro: Enter the project folder
          - npm install: Install dependencies
          - npm run dev: Start the application

      - Automatically recompiles code when files change

  ---------------------------------------------
  - Folder overview:
    - index.html: Main HTML file of the application

    - src: Source code of the React application
      - assets: Images and other static resources
      - App.jsx: Root component of the application
      - App.css: Styles related to App component
      - main.jsx: 
        - Entry point of the React application
        - Renders React appliation into the root element
      - index.css: Global styles

    - package.json: Metadata and dependency information of the project

    - node_modules: Dependency folders and files
    - public: Everything here is public and can be accessed by / file-name.ext

  ---------------------------------------------
  - Components:
    - Building block for React application
    - Small reusable pieces of UI
    - Features:
      - Can be reused
      - Can be nested
    - Function component syntax:
      function Component1() {
        return (
          <h1>Component 1</h1>
        )
      }
      export default Component1
      - Naming convention: PascalCase
      - Good practice: Keep filename and ComponentName same
    - Components should return "JSX"
    - To use a component in another file
      - Import the component
      - Syntax:
        <Component1 />
        <Component1></Component1>

    - JSX:
      - JavaScript XML Syntax
      - Allows writing HTML-like syntax inside JavaScript
      - Gets converted into JavaScript by React tooling
      - Rules:
        - Use className instead of class
          <div className='test'></div>
        - Self-closing tags either require / in the end, or a closing tag
          <img />
          <img></img>
